# 🎯 AI测试用例生成平台 - 功能更新

## ✨ 新增功能：在线文本输入测试用例生成

### 🚀 功能概述
现在支持**在线文本输入**直接生成测试用例，无需上传文档文件！用户可以直接在界面中输入功能描述，系统会智能分析并生成相应的测试用例。

---

## 🔥 主要特性

### 📝 双重输入模式
- **在线文本输入**：直接输入功能描述和详细内容
- **文档上传**：上传现有文档文件进行分析

### 🧠 智能场景识别
系统能够自动识别常见业务场景：
- **用户登录**：登录功能、认证流程
- **用户注册**：注册流程、数据验证
- **搜索功能**：关键词搜索、结果筛选
- **购买流程**：购物车、订单、支付
- **自定义场景**：用户自定义业务场景

### 🎯 快速模板
提供5个预设模板，一键应用：
- 用户登录功能
- 商品搜索功能
- 购物车功能
- 订单支付流程
- 用户注册功能

### 📊 增强的测试用例生成
- **基础功能验证**：核心功能测试
- **正向流程测试**：正常业务流程
- **异常处理验证**：边界条件和异常情况
- **性能测试**（可选）：并发和性能验证
- **补充测试**：确保测试覆盖完整

## 🛠️ 技术实现

### 后端增强
#### 新增API端点
```
POST /api/v1/testcases/generate-from-text
```

#### 参数支持
- `content`: 文本内容（必填）
- `test_type`: 测试类型（functional/performance/security）
- `scenario`: 测试场景描述（可选）

#### 智能生成算法
- 内容分析和关键词匹配
- 场景模板匹配
- 上下文感知生成
- 动态补充测试用例

### 前端改进
#### 界面优化
- 双模式切换（文本/文件）
- 功能描述输入框
- 详细内容文本域（支持2000字符）
- 快速模板按钮
- 拖拽上传支持

#### 用户体验
- 实时字数统计
- 表单验证提示
- 加载状态反馈
- 响应式设计

## 📖 使用指南

### 📝 在线文本输入方式
1. 打开测试用例生成页面
2. 选择"在线文本输入"模式
3. 填写功能描述（可选）
4. 输入详细的业务内容
5. 选择测试类型
6. 点击"生成测试用例"

### 📄 文档上传方式
1. 选择"上传文档"模式
2. 拖拽或点击上传文件
3. 选择测试类型
4. 点击"上传并生成"

### 🎯 快速模板使用
1. 点击任意模板按钮
2. 系统自动填充内容
3. 可根据需要修改内容
4. 直接生成测试用例
